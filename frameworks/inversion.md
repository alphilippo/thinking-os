# Inversion

## Quand l'utiliser

- Tu n'arrives pas à voir comment réussir → inverse en "comment échouer à coup sûr"
- Un plan semble solide mais tu sens que quelque chose cloche
- L'utilisateur dit "je veux X" mais la route vers X est encombrée
- Avant de lancer un projet : identifier les modes d'échec

## Procédure

1. **Noter l'objectif initial** : "Je veux atteindre X"
2. **Inverser** : "Comment garantir que je n'atteindrai PAS X ?"
3. **Lister exhaustivement** toutes les manières de foirer (sans filtre, même stupides) : 8-15 items.
4. **Pour chaque mode d'échec**, demander : "Est-ce que je suis en train de faire ça ?"
5. **Ranger** : les modes d'échec déjà en cours → priorité max. Les risques latents → surveillance.
6. **Convertir** en liste d'évitements actifs.

## Format synthèse

```
🧠 Framework : Inversion
📌 Pourquoi ce cadre : plus facile d'éviter la bêtise que de trouver le génie

Objectif : <X>
Question inversée : comment garantir de NE PAS atteindre X ?

Top 5 façons d'échouer :
1. <mode d'échec> → je le fais actuellement ? <oui/non/partiellement>
2. ...

Modes d'échec actifs chez toi : <liste>

✅ Action : arrêter IMMÉDIATEMENT <action destructrice>
⚠️ Angle mort : un mode d'échec silencieux qu'on n'a pas listé (ex : delay, inaction)
```

## Format deep-dive

1. **Objectif initial** reformulé précisément
2. **Inversion complète** : 10-15 modes d'échec (liste exhaustive sans filtre)
3. **Catégorisation** : échecs liés à la stratégie / l'exécution / la psychologie / l'environnement
4. **Auto-diagnostic** : pour chaque catégorie, lesquels sont actifs ?
5. **Hiérarchie d'arrêt** : quel comportement destructeur arrêter en premier ?
6. **Garde-fous** : quel signal détecte la rechute ?
7. **Inversion de l'inversion** : une fois tous les modes d'échec évités, qu'est-ce qu'il reste à faire activement ?

## Exemple bref

**Objectif** : *"Je veux que ma startup lève un tour de seed dans 6 mois"*

Inversion : comment GARANTIR de ne pas lever ?
1. Ne pas avoir de métriques de traction solides
2. Parler à trop peu d'investisseurs (<50)
3. Pitcher sans preuve de vente
4. Avoir un cap-table pourri (trop dilué early)
5. Ne pas savoir répondre à "pourquoi maintenant ?"
6. Construire en silo sans feedback marché
7. Dépenser le cash avant d'avoir levé
8. Mentir sur les chiffres (se fait cramer en DD)

Modes actifs détectés : #1 (pas de métriques claires), #3 (pas de vente), #5 (pas de réponse structurée).

✅ Action : avant de parler à UN investisseur, construire un dashboard de 5 métriques.
⚠️ Angle mort : échec #9 non listé → arriver trop tôt sur le marché.

## Pièges à éviter

- Ne pas confondre inversion et pessimisme. C'est un outil de clarté, pas de démoralisation.
- Ne pas juste lister les risques génériques ("la concurrence", "la régulation"). Être concret et actionnable.
- L'inversion ne remplace pas la stratégie offensive. Elle la complète.
- Si tu détectes qu'un mode d'échec est actif chez toi, dis-le directement — pas de sugar-coating.
